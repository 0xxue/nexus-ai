# AI QA System V3 — 企业级开源架构设计方案

> 开源项目。所有模型和服务可自由切换，不锁定任何服务商。
> 不需要任何 API Key 也能跑（Ollama 本地模型）。
> 标注 ⭐ 核心必做，🔜 后续迭代。

---

## 一、项目定位

**开源的企业内部 AI 智能问答平台**

- 员工用自然语言提问，系统自动查数据/文档/API，AI 生成分析+图表
- 支持上传文档（PDF/Word/Excel）建知识库
- 多轮对话，答案可溯源
- **完全开源，所有组件可自由切换，支持私有化部署**

**对标：** Dify / MaxKB / ChatBI，但更轻量、架构更现代

---

## 二、核心设计原则

| 原则 | 说明 |
|------|------|
| **模型自由** | LLM 和 Embedding 都通过 .env 切换，支持 Claude/GPT/DeepSeek/Ollama 本地/任何 OpenAI 兼容 API |
| **零成本可运行** | 用 Ollama 本地模型 + 本地 Embedding，不需要任何 API Key 也能跑 |
| **企业级架构** | LangGraph 编排 + 熔断器 + 链路追踪 + 限流 + 审计日志 |
| **可扩展** | 新增数据源只写一个 Adapter，新增模型改一行配置 |
| **生产就绪** | Docker 一键部署，Prometheus 监控，CI/CD |

## 三、模型自由切换方案

### LLM 模型（通过 LiteLLM 统一）

```env
# ===== 用户选择任意一种 =====

# 方案1：Claude（推理最强）
PRIMARY_MODEL=anthropic/claude-sonnet-4-20250514
ANTHROPIC_API_KEY=sk-ant-xxx

# 方案2：GPT（生态最大）
PRIMARY_MODEL=openai/gpt-4o
OPENAI_API_KEY=sk-xxx

# 方案3：DeepSeek（性价比最高）
PRIMARY_MODEL=deepseek/deepseek-chat
DEEPSEEK_API_KEY=sk-xxx

# 方案4：Ollama 本地（完全免费，不需要 API Key）
PRIMARY_MODEL=ollama/llama3
OLLAMA_BASE_URL=http://localhost:11434

# 方案5：任何 OpenAI 兼容 API（自建/第三方）
PRIMARY_MODEL=openai/custom-model
OPENAI_API_KEY=sk-xxx
OPENAI_API_BASE=https://your-api.com/v1

# ===== 可选：配置副模型做交叉验证 =====
SECONDARY_MODEL=deepseek/deepseek-chat
FALLBACK_MODEL=ollama/llama3
```

### Embedding 模型

```env
# 方案1：本地 sentence-transformers（默认，免费，不需要 GPU）
EMBEDDING_PROVIDER=local
EMBEDDING_MODEL=paraphrase-multilingual-MiniLM-L12-v2

# 方案2：Ollama 本地（免费，质量更高，需要 Ollama）
EMBEDDING_PROVIDER=ollama
EMBEDDING_MODEL=nomic-embed-text
OLLAMA_BASE_URL=http://localhost:11434

# 方案3：OpenAI（收费，质量最高）
EMBEDDING_PROVIDER=openai
EMBEDDING_MODEL=text-embedding-3-small
OPENAI_API_KEY=sk-xxx

# 方案4：任何 OpenAI 兼容 API
EMBEDDING_PROVIDER=openai_compatible
EMBEDDING_MODEL=BAAI/bge-m3
EMBEDDING_API_BASE=https://your-embedding-api.com/v1
EMBEDDING_API_KEY=xxx
```

### 代码实现

```python
# services/embedding.py — 根据配置自动选择 embedding 引擎
class EmbeddingService:
    def __init__(self, settings):
        match settings.embedding_provider:
            case "local":
                self.engine = LocalEmbedding(settings.embedding_model)
            case "ollama":
                self.engine = OllamaEmbedding(settings.ollama_base_url, settings.embedding_model)
            case "openai" | "openai_compatible":
                self.engine = OpenAIEmbedding(settings.embedding_api_base, settings.embedding_api_key, settings.embedding_model)

    async def embed(self, texts: list[str]) -> np.ndarray:
        return await self.engine.embed(texts)
```

## 五、核心功能

### ⭐ 1. 对话问答

```
用户提问 → 意图识别 → 数据源路由 → 多源并行获取 → AI 分析
→ 交叉验证 → 图表推荐 → 答案溯源 → SSE 流式返回
```

- 复合问题自动拆解（"最近一周到期数据，资金还能撑多久" → 2个子问题）
- 多数据源并行获取（asyncio.gather）
- 双模型交叉验证（置信度 < 0.7 时）
- 12 种 ECharts 图表
- 每条回答标注数据来源

### ⭐ 2. 知识库

- 上传 PDF / Word / Excel → 智能分块 → embedding → 向量索引
- 用户提问时 RAG 检索相关文档片段
- 答案引用标注来自哪个文档哪一页

### ⭐ 3. 多轮对话

- 对话列表 + 消息历史
- 上下文关联（最近 N 条消息作为 context）
- AI 自动生成对话标题

### ⭐ 4. 多数据源

| 数据源 | 路由条件 | 例子 |
|--------|---------|------|
| 内部 API | 实时数据查询 | "今天有多少用户" |
| 知识库 | 知识/政策/规章 | "报销流程是什么" |
| 数据库 | 报表/统计 | "上个月的收入报表" |

### ⭐ 5. 答案溯源

```json
{
    "answer": "最近一周共 23 个产品到期...",
    "sources": [
        {"type": "api", "name": "expiring_products", "time": "2025-12-01"},
        {"type": "document", "name": "Q3报告.pdf", "page": 12}
    ],
    "confidence": 0.85
}
```

### 🔜 6. 管理后台

模型配置 / 数据源管理 / 用户管理 / 审计日志 / 使用统计

### 🔜 7. 反馈机制

用户点赞/踩 → 持续优化

## 九、V2 功能迁移分析

### 被 V3 架构替代的（不需要搬）

| V2 功能 | V3 替代方案 | 省了多少代码 |
|---------|-----------|------------|
| 800 行编排器 | LangGraph 状态图 ~60 行 | -740 行 |
| 50+ 正则匹配 | LightRAG 语义搜索 | -300 行 |
| 双 Client（400 行） | LiteLLM 一行切模型 | -380 行 |
| AI 策略提取器 | LangGraph 条件边 | -200 行 |
| 查询类型检测（正则+同义词） | AI 直接识别意图 | -350 行 |
| 多层降级策略 | LiteLLM fallback + LangGraph fallback | -100 行 |
| API 调用去重器 | Redis 缓存层 | -80 行 |

### 必须保留的业务模块 ⭐

| 模块 | 说明 | 架构替代不了的原因 |
|------|------|------------------|
| **统一计算器** | 12 种精确计算（复利/ROI/现金跑道/趋势预测） | AI 算不准金融数据，需要 Decimal 高精度 |
| **财务格式化** | 货币(¥1,234.56)、百分比(↑12.5%)、趋势箭头、数值简化(1.2M) | 展示层逻辑 |
| **报告生成器** | 导出 Markdown/HTML/PDF，5 种主题 | 业务需求 |
| **图表服务** | 12 种图表 → ECharts JSON 配置（V2 是 Matplotlib 出图） | 可视化需求 |
| **时间序列构建器** | 缺失数据填补、趋势检测、异常值识别 | 数据处理逻辑 |
| **8 个业务 API** | 系统概览/日数据/到期产品/VIP分布/财务摘要等 | 业务接口 |
| **对话管理** | 多轮对话 + 历史 + 自动标题（用 SQLAlchemy 重建） | 基础功能 |
| **日期工具** | 简化版：格式化 + 范围计算（AI 处理"昨天""上周"） | 工具函数 |
